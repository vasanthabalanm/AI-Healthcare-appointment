# User Story - US_020

## Story ID
* US_020

## Story Title
* Patient joins waitlist for preferred unavailable slot

## Description
* As a patient, I want to register my interest in a specific unavailable slot on a waitlist, so that I am automatically offered that slot if it becomes available, without having to monitor availability manually.

## Acceptance Criteria

### AC-001: Patient successfully registers a new waitlist entry
- **Given**: A patient is authenticated, holds a confirmed appointment, and no active WaitlistEntry exists for them
- **When**: `POST /waitlist {patientId, preferredSlotId, confirmedAppointmentId}` is called for a future unavailable slot
- **Then**: HTTP 201 is returned with `{"waitlistId":"..."}`; a WaitlistEntry row is inserted with `status=Active`, `registeredAt=UtcNow`

### AC-002: Patient with existing active waitlist entry updates preferred slot
- **Given**: A patient already has a WaitlistEntry with `status=Active`
- **When**: `POST /waitlist` is called with a different `preferredSlotId`
- **Then**: HTTP 200 is returned with `{"message":"Waitlist entry updated"}`; the existing WaitlistEntry's `preferredSlotId` is updated; no duplicate entry is created

### AC-003: Waitlist entry for a past slot date returns 400
- **Given**: The `preferredSlotId` references a slot whose date is in the past or today before the current time
- **When**: `POST /waitlist` is called
- **Then**: HTTP 400 is returned with `{"error":"Preferred slot must be a future available slot date"}`; no WaitlistEntry is created

### AC-004: Unique partial index prevents two Active entries for same patient
- **Given**: A patient already has an Active WaitlistEntry
- **When**: A concurrent request attempts to INSERT a second Active WaitlistEntry for the same patient
- **Then**: SQL Server raises a unique constraint violation; the service returns HTTP 409; the existing Active entry is not affected

## Edge Cases
* Preferred slot becomes available between submission and INSERT — the slot is still unavailable at validation time (isAvailable=false check); the waitlist entry is created; the slot-swap job (us_021) handles the offer
* Patient cancels their confirmed appointment while on waitlist — WaitlistEntry should be reviewed; the service does not auto-delete the waitlist entry but flags it for review with a log warning
* Waitlist entry for a slot with zero remaining capacity — allowed; the slot-swap job will attempt booking and handle a capacity conflict at swap time

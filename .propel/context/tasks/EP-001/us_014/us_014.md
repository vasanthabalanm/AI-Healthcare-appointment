# User Story - US_014

## Story ID
* US_014

## Story Title
* RBAC policy enforcement at all API endpoints

## Description
* As a system operator, I want every API endpoint protected by server-side RBAC policies so that a Patient token cannot access Staff endpoints, a Staff token cannot access Admin endpoints, and any cross-role access attempt returns HTTP 403 and is logged to the audit trail.

## Acceptance Criteria

### AC-001: Patient token receives 403 on Staff-scoped endpoint
- **Given**: A JWT with `role=Patient` is present in the Authorization header
- **When**: `GET /staff/queue` is called
- **Then**: HTTP 403 is returned; an AuditLog entry is written with `actionType=READ`, `entityType=RBAC-Violation`, `actorRole=Patient`, and the requested path

### AC-002: Staff token receives 403 on Admin-scoped endpoint
- **Given**: A JWT with `role=Staff` is present
- **When**: `POST /admin/users` is called
- **Then**: HTTP 403 is returned; an AuditLog entry is written recording the violation

### AC-003: Admin token receives 403 on Patient-scoped endpoints marked as Patient-only
- **Given**: A JWT with `role=Admin` is present
- **When**: `POST /appointments` (Patient-only endpoint) is called
- **Then**: HTTP 403 is returned; the AuditLog entry is written

### AC-004: Correct-role token accesses endpoint successfully
- **Given**: A JWT with `role=Staff` is present and the endpoint requires `[Authorize(Policy="Staff")]`
- **When**: The request is made with a valid JWT
- **Then**: HTTP 200 (or appropriate success code) is returned; no RBAC violation AuditLog entry is written

### AC-005: Unauthenticated request to any protected endpoint returns 401
- **Given**: No Authorization header is present in the request
- **When**: Any non-public endpoint is called
- **Then**: HTTP 401 is returned; no AuditLog entry is written (unauthenticated requests are not attributed to an actor)

## Edge Cases
* Tampered JWT with modified role claim but valid signature — JWT signature validation rejects the token before role evaluation; HTTP 401 returned
* JWT with an unknown role value — policy evaluation treats unknown roles as unauthorised; HTTP 403 returned; structured log warning emitted
* Endpoint missing `[Authorize]` attribute accidentally — a startup convention test enumerates all controller actions and asserts every action has at least one `[Authorize]` or explicit `[AllowAnonymous]` attribute; CI fails if any action is unguarded

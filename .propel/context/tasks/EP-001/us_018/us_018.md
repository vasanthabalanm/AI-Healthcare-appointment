# User Story - US_018

## Story ID
* US_018

## Story Title
* HIPAA transmission security and PHI encryption in transit

## Description
* As a compliance officer, I want all API communications encrypted via TLS 1.2 or higher and all PHI at rest encrypted with AES-256, so that the platform satisfies HIPAA §164.312 technical safeguards for transmission security and access control.

## Acceptance Criteria

### AC-001: All HTTP requests are redirected to HTTPS
- **Given**: The IIS site is configured with an HTTPS binding and an HTTP redirect rule
- **When**: A client sends a plain HTTP request to any API endpoint
- **Then**: HTTP 301 is returned redirecting to the HTTPS equivalent URL; no API data is served over plain HTTP

### AC-002: TLS 1.2 minimum is enforced; TLS 1.0 and 1.1 are rejected
- **Given**: IIS is configured with the TLS cipher suite policy
- **When**: A TLS 1.1 or TLS 1.0 handshake is attempted
- **Then**: The TLS handshake fails with a protocol version error; no data is exchanged; the event is logged

### AC-003: HSTS header is present on all HTTPS responses from the Angular SPA
- **Given**: The Netlify/Vercel deployment includes an HSTS header configuration
- **When**: Any HTTPS response is received from the static host
- **Then**: The response includes `Strict-Transport-Security: max-age=31536000; includeSubDomains` header

### AC-004: PHI fields in API responses are never transmitted over non-TLS connections
- **Given**: An integration test spins up the API in test mode
- **When**: Any endpoint returning PHI (patient name, DOB, email, clinical fields) is called over plain HTTP
- **Then**: The request is redirected before any response body containing PHI is written; the test asserts the response body on the HTTP channel is empty

### AC-005: PHI encryption key is not present in any committed configuration file
- **Given**: All `appsettings.json`, `appsettings.*.json`, and any `.env` files committed to the repository
- **When**: A security scan checks for patterns matching AES key formats, connection strings, or JWT secrets
- **Then**: Zero matches are found; all sensitive values are loaded from environment variables or Windows DPAPI at runtime

## Edge Cases
* Certificate expiry during production — IIS should be configured with an automated certificate renewal (Let's Encrypt via Certbot or ACME client); a startup warning is logged when the certificate has fewer than 30 days remaining
* Mixed-content requests from Angular SPA — Angular `HttpClient` base URL uses `https://` exclusively in production; a Content-Security-Policy header blocks any mixed-content load
* TLS downgrade negotiation attempted — server cipher suite configuration removes all non-TLS 1.2/1.3 suites at the OS level (Windows IIS Crypto tool or Group Policy)

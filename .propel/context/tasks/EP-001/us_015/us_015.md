# User Story - US_015

## Story ID
* US_015

## Story Title
* JWT session management and 15-minute inactivity timeout

## Description
* As any authenticated user, I want my session to automatically expire after 15 minutes of inactivity with a 2-minute warning, so that unattended sessions cannot be exploited and I can extend my session with a single action before it expires.

## Acceptance Criteria

### AC-001: Login issues a JWT and stores token in Redis allowlist
- **Given**: Valid credentials are provided to `POST /auth/login`
- **When**: Authentication succeeds
- **Then**: HTTP 200 is returned with `{"token":"<jwt>","role":"<role>"}`; the JWT has `exp` set to 15 minutes from now; a Redis entry `SET token:{jti} {userId} EX 900` is created; the response includes a role-based dashboard redirect hint

### AC-002: Valid Redis-backed token allows authenticated request and resets TTL
- **Given**: A user holds a JWT whose `jti` exists in Redis
- **When**: Any authenticated request is made
- **Then**: Redis `GET token:{jti}` returns the userId; Redis `EXPIRE token:{jti} 900` resets the TTL; the request succeeds with the appropriate response

### AC-003: Token absent from Redis (expired or revoked) returns 401
- **Given**: A JWT's `jti` is not present in Redis (TTL elapsed or explicit logout)
- **When**: An authenticated request is made with that JWT
- **Then**: HTTP 401 is returned regardless of JWT signature validity; the session is not extended; the user must re-authenticate

### AC-004: Session extension resets the 15-minute TTL
- **Given**: A user holds a valid session with < 2 minutes remaining (TTL < 120 seconds)
- **When**: `POST /auth/extend-session` is called with the current Bearer token
- **Then**: Redis `EXPIRE token:{jti} 900` is called; HTTP 200 is returned with `{"message":"Session extended"}`; the next request timeout is 15 minutes from this call

### AC-005: Logout removes the token from Redis immediately
- **Given**: A user is authenticated and calls `POST /auth/logout`
- **When**: The request is processed
- **Then**: Redis `DEL token:{jti}` is executed; HTTP 200 is returned; subsequent requests with the same JWT receive HTTP 401

### AC-006: Account lockout after 5 failed login attempts
- **Given**: 5 consecutive failed login attempts have been made for an account
- **When**: A 6th login attempt is made within the lockout window
- **Then**: HTTP 401 is returned with `{"error":"Account temporarily locked — try again in 15 minutes"}`; the lock expires automatically after 15 minutes

## Edge Cases
* Redis connection lost during token validation — falls through to database-backed fallback check; if no fallback available, returns HTTP 503 with a structured log critical error (not a silent pass-through)
* Multiple concurrent requests during the 2-minute warning window — all reset the TTL independently; no race condition causes an accidental logout
* JWT issued with a future `iat` (clock skew attack) — JWT validation rejects tokens with `iat` more than 30 seconds in the future

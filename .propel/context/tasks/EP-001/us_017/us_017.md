# User Story - US_017

## Story ID
* US_017

## Story Title
* Password reset via time-limited email token

## Description
* As a registered user, I want to request a password reset email and set a new password via a one-time token, so that I can regain access to my account securely without requiring Admin intervention.

## Acceptance Criteria

### AC-001: Password reset request sends email with 60-minute token
- **Given**: A UserAccount with the provided email exists and is Active
- **When**: `POST /auth/forgot-password {email}` is called
- **Then**: HTTP 200 is returned with `{"message":"If that email is registered, a reset link has been sent"}`; a single-use reset token is generated and emailed; the token expires in 60 minutes; the response is identical whether the email exists or not (prevents email enumeration)

### AC-002: Valid token within 60 minutes resets the password
- **Given**: A valid reset token has been issued and is within the 60-minute window
- **When**: `POST /auth/reset-password {token, newPassword}` is called
- **Then**: HTTP 200 is returned; the `PasswordHash` is updated using PBKDF2 SHA-256 100k iterations; the token is marked as used; any active Redis session tokens for the user are revoked (DEL all `token:{jti}` entries for the user)

### AC-003: Expired reset token returns 400
- **Given**: A reset token was issued more than 60 minutes ago
- **When**: `POST /auth/reset-password {token, newPassword}` is called
- **Then**: HTTP 400 is returned with `{"error":"Password reset token has expired"}`; the password is not changed

### AC-004: Already-used token returns 400
- **Given**: A reset token has already been used to reset the password
- **When**: The same token is submitted again
- **Then**: HTTP 400 is returned with `{"error":"Password reset token has already been used"}`; the password is not changed again

### AC-005: New password that fails complexity validation returns 422
- **Given**: A valid reset token is submitted with a `newPassword` that does not meet complexity requirements (e.g., < 8 characters, no uppercase)
- **When**: `POST /auth/reset-password` is called
- **Then**: HTTP 422 is returned with field-level validation errors; the password is not updated; the token remains valid for retry

## Edge Cases
* Password reset request for unknown email — always returns HTTP 200 with the same generic message to prevent email enumeration
* Concurrent password reset requests for same account — only the most recently issued token is valid; earlier tokens are invalidated on new request issuance
* Reset link clicked in an email client that pre-fetches link previews — GET-safe preview URL design: the actual password change requires a POST with both token and new password, preventing accidental reset from link preview

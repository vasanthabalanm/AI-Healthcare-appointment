# User Story - US_016

## Story ID
* US_016

## Story Title
* Immutable audit log with HTTP 405 modification guard

## Description
* As a compliance officer, I want every authenticated action recorded in an append-only audit log that cannot be modified or deleted by any user or API endpoint, so that the platform maintains a tamper-proof record of all data access and changes as required by HIPAA §164.312.

## Acceptance Criteria

### AC-001: Every CRUD action by any authenticated actor generates an AuditLog entry
- **Given**: An authenticated user performs a CREATE, READ, UPDATE, or DELETE operation on any entity
- **When**: The operation completes (success or failure)
- **Then**: An AuditLog entry is inserted with the correct `actorId`, `actorRole`, `actionType`, `entityType`, `entityId`, `beforeValue` (for updates/deletes), `afterValue` (for creates/updates), `ipAddress`, and `timestamp`; the insert uses the INSERT-only database role

### AC-002: Admin-initiated audit log export returns paginated results
- **Given**: An Admin calls `GET /audit?dateFrom=&dateTo=&actor=&action=&page=1`
- **When**: The query is processed
- **Then**: HTTP 200 is returned with `{"entries":[...],"pagination":{"page":1,"total":...}}`; entries are ordered by `timestamp DESC`; maximum 50 entries per page

### AC-003: Audit log export of ≤10,000 rows returns CSV immediately
- **Given**: The filter matches ≤10,000 audit rows
- **When**: `GET /audit/export?dateFrom=&dateTo=` is called
- **Then**: HTTP 200 is returned with a CSV file attachment; the file contains all matching rows with headers

### AC-004: Audit log export >10,000 rows is queued asynchronously
- **Given**: The filter matches >10,000 audit rows
- **When**: `GET /audit/export?dateFrom=&dateTo=` is called
- **Then**: HTTP 202 is returned with `{"message":"Export queued — you will be notified when ready"}`; a Hangfire job is enqueued to generate and email the export

### AC-005: DELETE or PATCH on audit entries returns HTTP 405 and logs the attempt
- **Given**: Any authenticated user calls `DELETE /audit/{id}` or `PATCH /audit/{id}`
- **When**: The request reaches the API
- **Then**: HTTP 405 is returned with `{"error":"Audit log entries are immutable"}`; the attempt itself is logged as an AuditLog entry with `actionType=DELETE` and a `modificationAttempt=true` flag

## Edge Cases
* AuditLog insert fails during a business operation — the audit write must not be in the same SaveChanges transaction as the business entity; a separate fire-and-forget INSERT ensures audit entries are captured even when business writes succeed
* Bulk data export times out — CSV generation for large datasets uses streaming (IAsyncEnumerable) to avoid loading all rows into memory; timeout set to 120 seconds for sync export
* AuditLog table approaches SQL Server Express 10 GB limit — a startup check warns at 80% of database size; audit archival to flat files should be planned before limit is reached

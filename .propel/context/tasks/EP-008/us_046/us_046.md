# User Story - US_046

## Story ID
* US_046

## Story Title
* CPT procedure code generation via Ollama BioMistral

## Description
* As a clinician, I want CPT procedure codes to be automatically suggested for a patient's verified clinical record using a separate LLM prompt template, so that procedure billing codes are available for Staff review alongside ICD-10 diagnosis codes.

## Acceptance Criteria

### AC-001: CPT code generation uses a distinct prompt template from ICD-10
- **Given**: `POST /patients/{id}/generate-codes {type:"CPT"}` is called with a Verified patient record
- **When**: The Hangfire CPT generation job runs
- **Then**: The Ollama request uses the `CPT_PROMPT_TEMPLATE` (distinct from `ICD10_PROMPT_TEMPLATE`); the LLM is explicitly asked to return CPT codes, not ICD-10

### AC-002: CPT suggestions are stored separately from ICD-10 suggestions
- **Given**: Both ICD-10 and CPT generation have been triggered for a patient
- **When**: Suggestions are stored in PostgreSQL
- **Then**: `MedicalCodeSuggestion` rows with `codeType=ICD10` and `codeType=CPT` are separate rows; no mixing of code types in a single row; the Staff coding UI distinguishes the two lists

### AC-003: "No procedures identified" response inserts no CPT suggestions
- **Given**: The patient's clinical record contains diagnoses but no identifiable procedure descriptions
- **When**: BioMistral returns "No procedures identified"
- **Then**: Zero `MedicalCodeSuggestion` rows are inserted for CPT; no zero-confidence placeholder rows are created; the coding dashboard shows "No CPT suggestions available"

### AC-004: CPT code format validation rejects non-CPT strings
- **Given**: BioMistral returns a string that does not match CPT format `^\d{5}$`
- **When**: The job processes the response
- **Then**: The malformed code is rejected and not inserted; the rejection is logged as a WARNING

## Edge Cases
* BioMistral returns both ICD-10 and CPT codes in a CPT prompt response (model confusion) — only rows where the code format matches CPT regex `^\d{5}$` are inserted with `codeType=CPT`; ICD-10 format codes in a CPT response are logged as unexpected and discarded
* Ollama responds with a narrative explanation instead of a code list — the response parser extracts codes from the narrative using regex; if no codes are found, "No CPT suggestions available" state is set
* CPT code generation triggered when ICD-10 generation is already in progress — both jobs run independently; Hangfire job concurrency is handled by separate job IDs

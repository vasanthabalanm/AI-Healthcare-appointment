# User Story - US_048

## Story ID
* US_048

## Story Title
* AI-Human Agreement Rate metric for Admin dashboard

## Description
* As an Admin, I want to see a rolling 30-day AI-Human Agreement Rate on the Admin dashboard, so that I can monitor the quality of AI medical code suggestions and make informed decisions about model confidence thresholds or Staff training needs.

## Acceptance Criteria

### AC-001: GET agreement-rate endpoint returns the rolling 30-day rate
- **Given**: MedicalCodeSuggestion rows with `status IN (Accepted, Modified, Rejected)` exist within the past 30 days
- **When**: `GET /admin/metrics/code-agreement?days=30` is called by an Admin
- **Then**: HTTP 200 is returned with `{"agreementRate":0.84,"totalActioned":250,"accepted":210,"modified":25,"rejected":15,"windowDays":30}`

### AC-002: Agreement rate calculation is: accepted where committedCode == suggestedCode / total actioned
- **Given**: 100 suggestions have been actioned in 30 days: 80 accepted with no change, 10 accepted with modified code, 10 rejected
- **When**: The metric is calculated
- **Then**: `agreementRate = 80 / 100 = 0.80`; Modified actions count as disagreements; Rejected actions count as disagreements; only exact matches between `committedCode` and `suggestedCode` count as agreements

### AC-003: Endpoint is restricted to Admin role only
- **Given**: A JWT with `role=Staff` or `role=Patient` is present
- **When**: `GET /admin/metrics/code-agreement` is called
- **Then**: HTTP 403 is returned; the metrics data is not exposed

### AC-004: Zero actioned suggestions in the window returns a defined response
- **Given**: No suggestions have been actioned in the last 30 days
- **When**: `GET /admin/metrics/code-agreement?days=30` is called
- **Then**: HTTP 200 is returned with `{"agreementRate":null,"totalActioned":0,"message":"No code suggestions actioned in the past 30 days"}`; division by zero is avoided

### AC-005: Custom day window is supported via query parameter
- **Given**: `GET /admin/metrics/code-agreement?days=7` is called
- **When**: The metric is calculated
- **Then**: Only suggestions actioned within the past 7 days are included; the response includes `"windowDays":7`; the default is 30 days if the parameter is omitted

## Edge Cases
* `days` parameter is 0 or negative — HTTP 422 returned with `{"error":"days parameter must be a positive integer"}`
* `days` parameter exceeds 365 — capped at 365 with a WARNING log and `"windowDays":365` in the response
* Agreement rate hovers near 0% — Admin receives no automated alert from this endpoint; the metric is informational only; alerting thresholds are a future enhancement

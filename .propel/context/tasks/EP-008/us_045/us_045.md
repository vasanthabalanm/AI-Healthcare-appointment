# User Story - US_045

## Story ID
* US_045

## Story Title
* ICD-10 code generation via Ollama BioMistral 7B Hangfire job

## Description
* As a clinician, I want ICD-10 diagnosis codes to be automatically suggested for a patient's verified clinical record by a local BioMistral LLM, so that medical coding is accelerated while remaining fully under Staff review before any code is committed.

## Acceptance Criteria

### AC-001: ICD-10 generation is triggered only when the patient record is Verified
- **Given**: A Staff member calls `POST /patients/{id}/generate-codes {type:"ICD10"}`
- **When**: The request is processed
- **Then**: If `verificationStatus != Verified`, HTTP 409 is returned with `{"error":"Verify patient record before generating codes"}`; if Verified, HTTP 202 is returned with `{"message":"ICD-10 code generation queued","jobId":"..."}`; a Hangfire job is enqueued

### AC-002: Hangfire job calls Ollama BioMistral and inserts MedicalCodeSuggestion rows as Pending
- **Given**: The Hangfire ICD-10 generation job runs
- **When**: `POST localhost:11434/api/chat` is called with the structured 360° view summary as prompt
- **Then**: Each code in the LLM response is inserted as a `MedicalCodeSuggestion` row with `codeType=ICD10`, `status=Pending`, `verifiedById=null`, `suggestedCode`, `confidence_score`

### AC-003: Low-confidence LLM output is flagged for Staff attention
- **Given**: The LLM returns a code with `confidence_score < 0.60`
- **When**: The code is inserted
- **Then**: `MedicalCodeSuggestion.lowConfidenceFlag = true`; the Staff dashboard highlights low-confidence suggestions separately

### AC-004: Hangfire job retries on Ollama timeout with exponential backoff
- **Given**: Ollama at `localhost:11434` does not respond within 30 seconds
- **When**: The job times out
- **Then**: Retry at ~30s, ~60s, ~120s; after 3 failures the job moves to dead-letter; `MedicalCodeSuggestion` rows are not inserted until a retry succeeds; an "engine unavailable" notification is sent to Staff

### AC-005: Dead-letter job surfaces as "engine unavailable" status on the coding dashboard
- **Given**: All retries are exhausted
- **When**: Staff views the coding dashboard for the patient
- **Then**: The coding status shows "AI engine unavailable — manual coding required"; no partial Pending suggestions remain from failed attempts

## Edge Cases
* Ollama returns more than 20 ICD-10 codes — the job accepts up to 20 suggestions; remaining codes are silently dropped with a DEBUG log entry noting truncation
* Patient record updated after code generation is queued but before job runs — the job reads the 360° view at execution time, not at enqueue time; the most current verified record is used
* BioMistral response contains non-ICD-10 format codes — regex validation `[A-Z][0-9]{2}(\.[0-9]{1,4})?` rejects malformed codes; the rejection count is logged as a WARNING

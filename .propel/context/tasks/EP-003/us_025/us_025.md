# User Story - US_025

## Story ID
* US_025

## Story Title
* Microsoft Outlook Calendar OAuth2 PKCE appointment sync

## Description
* As a patient, I want to add my confirmed appointment to my Microsoft Outlook Calendar with a single action using Microsoft Graph API, so that I receive Outlook calendar reminders without manually entering the details.

## Acceptance Criteria

### AC-001: Microsoft Graph OAuth2 PKCE flow initiates and redirects to Microsoft consent screen
- **Given**: A patient calls `POST /appointments/{id}/calendar-sync {provider:"outlook"}`
- **When**: The request is processed
- **Then**: HTTP 302 is returned redirecting to the Microsoft identity platform OAuth consent URL; the URL includes `code_challenge`, `code_challenge_method=S256`, and `scope=https://graph.microsoft.com/Calendars.ReadWrite`

### AC-002: Successful Microsoft callback creates Outlook calendar event
- **Given**: The patient grants Calendars.ReadWrite permission and is redirected to `GET /calendar/callback`
- **When**: The code is exchanged via Microsoft token endpoint
- **Then**: Access and refresh tokens are stored encrypted (AES-256) per patient; a Microsoft Graph `POST /me/events` call creates the calendar event with appointment summary, start (dateTimeTimeZone), end, and location; HTTP 200 is returned with `{"message":"Added to Outlook Calendar"}`

### AC-003: Patient denies Microsoft consent — graceful cancellation
- **Given**: The patient clicks "Cancel" on the Microsoft consent screen
- **When**: The redirect returns with `error=access_denied`
- **Then**: HTTP 200 is returned with `{"message":"Calendar sync cancelled — no event created"}`; no tokens are stored

### AC-004: Microsoft Graph API rate limit or error returns 503
- **Given**: Microsoft Graph returns HTTP 429 or 503 during event creation
- **When**: The Graph API call is made
- **Then**: HTTP 503 is returned to the patient with `{"error":"Calendar sync failed — retry available"}`; error is logged

## Edge Cases
* Outlook calendar event creation fails due to missing `dateTimeTimeZone` format — Graph API requires RFC 3339 format with explicit timezone; the service converts appointment UTC time to the patient's configured timezone (defaulting to UTC if not set)
* Microsoft tenant policy blocks third-party app consent — the OAuth flow fails; patient receives "Your organisation's admin has restricted calendar sync" message
* Duplicate event creation (user clicks sync twice) — idempotency check using Microsoft Graph `$filter` before creation; if an event with the same appointment ID exists, return 200 without creating a duplicate

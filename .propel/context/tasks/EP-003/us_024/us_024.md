# User Story - US_024

## Story ID
* US_024

## Story Title
* Google Calendar OAuth2 PKCE appointment sync

## Description
* As a patient, I want to add my confirmed appointment to my Google Calendar with a single action, so that I receive calendar reminders automatically without manually entering the appointment details.

## Acceptance Criteria

### AC-001: OAuth2 PKCE flow initiates and redirects to Google consent screen
- **Given**: A patient calls `POST /appointments/{id}/calendar-sync {provider:"google"}`
- **When**: The request is processed
- **Then**: HTTP 302 is returned redirecting to the Google OAuth consent URL; the URL includes `code_challenge` and `code_challenge_method=S256` (PKCE); `state` contains a signed appointment-to-session binding

### AC-002: Successful OAuth callback exchanges code and creates calendar event
- **Given**: The patient grants calendar write permission and is redirected to `GET /calendar/callback {code, state}`
- **When**: The code is exchanged for tokens via `POST /token`
- **Then**: Access and refresh tokens are stored encrypted (AES-256) per patient in SQL Server; a Google Calendar event is created via `POST /calendars/primary/events` with appointment summary, start, end, and location; HTTP 200 is returned with `{"message":"Added to Google Calendar"}`

### AC-003: Patient denies OAuth consent — graceful cancellation
- **Given**: The patient clicks "Deny" on the Google consent screen
- **When**: Google redirects back with `error=access_denied`
- **Then**: HTTP 200 is returned with `{"message":"Calendar sync cancelled — no event created"}`; no tokens are stored; no calendar event is created

### AC-004: Google Calendar API rate limit or timeout returns 503 with retry hint
- **Given**: Google Calendar API returns HTTP 429 or a timeout occurs
- **When**: The event creation request is made
- **Then**: HTTP 503 is returned to the patient with `{"error":"Calendar sync failed — retry available"}`; the error is logged with correlation ID; no partial state is stored

## Edge Cases
* PKCE `state` parameter tampered (CSRF attempt) — the callback handler validates the signed state parameter; mismatch returns HTTP 400 and logs a security warning
* OAuth refresh token expired — the next sync attempt detects the expired refresh token; patient is prompted to re-authorise; existing calendar events are not affected
* Patient has no Google account — the OAuth flow fails at consent with a Google-side error; the application surfaces a friendly "Google account required" message

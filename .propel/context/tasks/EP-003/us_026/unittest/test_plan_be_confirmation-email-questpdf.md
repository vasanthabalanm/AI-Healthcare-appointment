## Test Plan Overview

Covers backend unit tests for the Hangfire background job that sends a booking confirmation email with a QuestPDF-generated attachment:

1. **`ConfirmationPdfGenerator.Generate(AppointmentConfirmationDto dto)`** â€” pure static method that produces a `byte[]` PDF; directly testable without SMTP.
2. **`SendConfirmationEmailJob.ExecuteAsync(int appointmentId, IJobCancellationToken cancellationToken)`** â€” Hangfire job that loads appointment data, generates PDF, builds MIME message, and sends via MailKit SMTP.

> **SMTP Testability Constraint:** `SendConfirmationEmailJob` creates `new SmtpClient()` directly (no interface injection). Tests that need to reach the SMTP step must use the `Environment.SetEnvironmentVariable("SMTP_HOST", "127.0.0.1")` trick â€” the job will throw on `ConnectAsync` (no real SMTP server), and assertions target the observable state before the throw. Tests for guard paths (null appointment, null patient/slot) do NOT need SMTP env vars because the method returns before reaching that code.

> **NEW FILE REQUIRED:** `tests/ClinicalHealthcare.Infrastructure.Tests/Jobs/SendConfirmationEmailJobTests.cs` **does not exist** and must be created as part of this test plan.

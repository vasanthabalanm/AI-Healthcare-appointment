# User Story - US_026

## Story ID
* US_026

## Story Title
* Booking confirmation email with QuestPDF attachment within 60 seconds

## Description
* As a patient, I want to receive a confirmation email with a PDF attachment containing my appointment details within 60 seconds of booking, so that I have an immediate, portable record of my appointment.

## Acceptance Criteria

### AC-001: Confirmation email with PDF attachment is dispatched within 60 seconds of booking
- **Given**: A patient successfully books an appointment via `POST /appointments`
- **When**: The booking is confirmed and the Hangfire job is enqueued
- **Then**: The confirmation email arrives at the patient's verified email address within 60 seconds; the email contains a PDF attachment; the PDF includes appointment ID, date, time, location, and provider name

### AC-002: QuestPDF generates the PDF in under 2 seconds
- **Given**: An appointment confirmation PDF generation is triggered
- **When**: `QuestPDF.Fluent.Document.Create()` generates the document
- **Then**: The PDF byte stream is ready within 2000 ms; the PDF is streamed directly as a MailKit email attachment without any intermediate filesystem write

### AC-003: Email is sent via SMTPS port 465 with credentials from environment variables
- **Given**: MailKit SMTP client is configured with `UseSsl=true, Port=465`
- **When**: The confirmation email is sent
- **Then**: The SMTP connection uses TLS (SMTPS); SMTP username and password are read from environment variables; no credentials appear in any log output

### AC-004: Hangfire job retries on transient SMTP failure with exponential backoff
- **Given**: The SMTP server is temporarily unavailable when the job first executes
- **When**: The Hangfire job runs
- **Then**: The job is retried at ~30 s, ~60 s, and ~120 s; after 3 retries the job moves to dead-letter queue; a Staff dashboard alert is raised for the failed confirmation

### AC-005: PDF generation failure sends email without attachment rather than failing completely
- **Given**: QuestPDF throws an exception during document generation
- **When**: The Hangfire notification job handles the exception
- **Then**: The exception is caught; the email is sent without the PDF attachment with a note "PDF attachment temporarily unavailable"; the failure is logged as a structured warning with correlation ID

## Edge Cases
* Patient email address changes between booking and job execution — the Hangfire job reads the email from the UserAccount at execution time, not at enqueue time; if the email changed, the new address receives the confirmation
* PDF size exceeds SMTP attachment limit — the template is designed for a single appointment; size should not exceed 200 KB; a size assertion is included in the unit test
* SMTP credentials rotate in production — updating environment variables requires an application restart to pick up new values; this is documented in the operations runbook

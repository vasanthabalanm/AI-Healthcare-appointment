# User Story - US_027

## Story ID
* US_027

## Story Title
* Automated SMS appointment reminders via ISmsGateway

## Description
* As a patient, I want to receive automated SMS reminders at 48 hours and 2 hours before my appointment, so that I am less likely to forget my appointment and can take timely action to cancel if needed.

## Acceptance Criteria

### AC-001: T-48h SMS reminder is sent 48 hours before appointment time
- **Given**: An appointment is scheduled and the T-48h Hangfire reminder job is enqueued at booking time
- **When**: The scheduled job fires 48 hours before the appointment
- **Then**: An SMS is sent to the patient's registered phone number via `ISmsGateway.SendAsync()`; the message contains the appointment date, time, and location; no plain-text PHI beyond appointment logistics is included

### AC-002: T-2h SMS reminder is sent 2 hours before appointment time
- **Given**: The T-2h Hangfire reminder job is scheduled
- **When**: The job fires 2 hours before appointment time
- **Then**: An SMS is sent with the same content format as the T-48h reminder

### AC-003: Twilio/Vonage sandbox SMS gateway receives the request
- **Given**: The `ISmsGateway` implementation is configured with a Twilio or Vonage sandbox API key from environment variables
- **When**: `SendAsync(to, message)` is called
- **Then**: The gateway API receives the POST request with the correct to-number and message body; the request uses HTTPS; credentials are sourced from environment variables

### AC-004: SMS gateway failure triggers Hangfire retry with exponential backoff
- **Given**: The SMS gateway returns an HTTP 5xx error or times out
- **When**: The Hangfire job fails
- **Then**: The job is retried at ~30 s, ~60 s, and ~120 s; after 3 retries the job moves to dead-letter; the email reminder proceeds independently

### AC-005: Cancelled appointment's pending SMS reminder jobs are cancelled
- **Given**: An appointment is cancelled by the patient
- **When**: The cancellation processing completes
- **Then**: Both pending T-48h and T-2h SMS reminder Hangfire jobs are deleted from the queue; no SMS is sent for the cancelled appointment

## Edge Cases
* Patient has no phone number on file — SMS job skips gracefully with a structured log info entry; email reminder proceeds; no exception thrown
* Phone number in non-E.164 format — normalisation step applied before gateway call; if normalisation fails, the job logs a warning and skips SMS without failing the whole reminder pipeline
* Gateway credentials rotated — updated environment variables require application restart; operations runbook documents this procedure

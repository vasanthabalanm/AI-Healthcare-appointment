# User Story - US_040

## Story ID
* US_040

## Story Title
* OCR text extraction from clinical documents via Tesseract 5.x Hangfire job

## Description
* As a clinician, I want clinical document images to be automatically processed by OCR in the background after upload, so that the text content of scanned documents becomes available for structured extraction without manual transcription.

## Acceptance Criteria

### AC-001: Hangfire OCR job decrypts blob, runs Tesseract, stores raw text result
- **Given**: A ClinicalDocument exists with `extractionStatus=Pending`
- **When**: The OCR Hangfire job executes
- **Then**: The encrypted blob is decrypted to an in-memory buffer; Tesseract 5.x P/Invoke processes the buffer; the raw extracted text is stored in the `ClinicalDocument.RawOcrText` field; the in-memory buffer is disposed immediately after Tesseract completes

### AC-002: Confidence ≥ 0.75 sets extraction status to Extracted
- **Given**: Tesseract returns an overall confidence score ≥ 0.75
- **When**: The job processes the extraction result
- **Then**: `ClinicalDocument.extractionStatus` is updated to `Extracted`; a Hangfire field-extraction job is enqueued to process the raw text

### AC-003: Confidence < 0.75 sets extraction status to LowConfidence
- **Given**: Tesseract returns a confidence score < 0.75
- **When**: The job processes the result
- **Then**: `extractionStatus` is updated to `LowConfidence`; the raw text is stored for manual review; a Staff alert is raised; the field-extraction job is not enqueued

### AC-004: No readable text detected sets status to NoData
- **Given**: Tesseract returns empty text or confidence = 0 (e.g., blank page or purely graphical image)
- **When**: The job processes the result
- **Then**: `extractionStatus` is updated to `NoData`; no field-extraction job is enqueued; Staff is notified that manual review is required

### AC-005: Job failure triggers exponential backoff retry 3 times then dead-letter
- **Given**: The Tesseract P/Invoke call throws an exception (e.g., native library crash)
- **When**: The Hangfire job fails
- **Then**: Retry is attempted at ~30s, ~60s, ~120s; after 3 failures the job is moved to dead-letter queue; `extractionStatus` remains `Pending` until a retry succeeds or manual resolution occurs; a CRITICAL log entry is written

## Edge Cases
* Multi-page PDF — Tesseract processes each page sequentially; overall confidence = average of per-page confidence scores; text is concatenated in page order
* Encrypted blob deleted from disk before job runs — job logs a CRITICAL error with the document ID; `extractionStatus` is set to `Failed`; document metadata remains in the database
* Tesseract native library version mismatch — application startup validates the Tesseract library version; a startup exception is thrown if < 5.0.0

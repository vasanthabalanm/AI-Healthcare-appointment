# User Story - US_041

## Story ID
* US_041

## Story Title
* Structured clinical field extraction from OCR text

## Description
* As a clinician, I want OCR-extracted text to be automatically parsed into structured clinical fields such as vital signs, medications, and diagnoses, so that the data becomes machine-readable and usable for the 360° patient view without manual data entry.

## Acceptance Criteria

### AC-001: NLP pipeline maps OCR text to structured clinical field types
- **Given**: A ClinicalDocument has `extractionStatus=Extracted` and `RawOcrText` is populated
- **When**: The Hangfire field-extraction job runs
- **Then**: The raw OCR text is parsed by the clinical NLP pipeline; for each recognised segment, an `ExtractedClinicalField` is inserted with `fieldType` set to one of `VitalSign`, `MedicalHistory`, `Medication`, `Allergy`, or `Diagnosis`

### AC-002: Confidence score is propagated from OCR confidence to extracted fields
- **Given**: The OCR extraction produced a confidence of 0.85
- **When**: ExtractedClinicalField rows are inserted
- **Then**: Each field's `confidence_score` is set to the segment-level confidence (or the document-level confidence if segment-level is not available); no field is inserted with `confidence_score = 0.0`

### AC-003: Extracted fields reference the correct patient and document IDs
- **Given**: The document belongs to `patientId=A`
- **When**: ExtractedClinicalField rows are inserted
- **Then**: Each row has `patient_id=A` and `document_id` referencing the source ClinicalDocument; no orphaned fields with null patient references

### AC-004: Unrecognised text segments are not inserted as Unknown type fields
- **Given**: A portion of the OCR text does not match any clinical field pattern
- **When**: The NLP pipeline processes the text
- **Then**: No ExtractedClinicalField is inserted for unrecognised segments; unmatched text is logged at DEBUG level for diagnostics

## Edge Cases
* Same medication appears multiple times in one document — each occurrence is inserted as a separate ExtractedClinicalField; de-duplication is handled separately in the aggregation job (us_042)
* OCR text contains PHI in free text (e.g., patient name in notes) — the NLP pipeline is not responsible for PHI redaction; all extracted field values are stored as-is; access is controlled by RBAC at the API layer
* Field value exceeds text column maximum — field value truncated at 2000 characters with a WARNING log; the full value is available in the document's RawOcrText

# User Story - US_038

## Story ID
* US_038

## Story Title
* Patient clinical document upload with ClamAV virus scan gate

## Description
* As a patient, I want to securely upload clinical documents (PDF only, ≤10MB) and have them scanned for viruses before storage, so that my health records are safely ingested into the system without exposing the platform to malicious files.

## Acceptance Criteria

### AC-001: Valid PDF within size limit passes virus scan and is encrypted at rest
- **Given**: A patient uploads a valid PDF ≤ 10 MB via `POST /documents/upload`
- **When**: The file is received
- **Then**: The file is scanned via nClam TCP connection to the ClamAV daemon; scan result = Clean; file is encrypted with AES-256-CBC and written to the configured blob path; a ClinicalDocument row is inserted with `virusScanResult=Pass`, `extractionStatus=Pending`; an OCR Hangfire job is enqueued; HTTP 201 is returned

### AC-002: Infected file is rejected with 422 and not written to disk
- **Given**: ClamAV returns a virus-found result for the uploaded file
- **When**: The scan completes
- **Then**: HTTP 422 is returned with `{"error":"File rejected — virus detected"}`; no file is written to the blob storage path; no ClinicalDocument row is inserted; the infection event is logged as CRITICAL

### AC-003: ClamAV daemon unreachable returns 503 — upload is rejected
- **Given**: The ClamAV `clamd` Windows Service is not running or the TCP connection times out
- **When**: A file upload is attempted
- **Then**: HTTP 503 is returned with `{"error":"Virus scan service unavailable — upload cannot proceed"}`; the file is not written to disk under any circumstances; silent bypass is not permitted

### AC-004: OCR extraction is asynchronous — upload response does not wait for OCR
- **Given**: A clean file is accepted and encrypted
- **When**: HTTP 201 is returned to the patient
- **Then**: The Hangfire OCR job is enqueued but not yet executed; the response does not include OCR results; the patient can query extraction status later via `GET /documents/{id}/status`

### AC-005: Non-PDF file type returns 415
- **Given**: A patient uploads a `.docx` or `.jpg` file
- **When**: The upload endpoint validates the MIME type
- **Then**: HTTP 415 is returned with `{"error":"Only PDF files are accepted"}`; no scan or storage occurs

## Edge Cases
* File exactly at 10 MB limit — accepted; file slightly over 10 MB returns HTTP 413 before the virus scan runs
* Concurrent uploads from same patient — each upload creates a separate ClinicalDocument with a unique ID and blob path; no race condition on file paths
* ClamAV returns INSTREAM error (malformed stream) — treated as an inconclusive scan result; file is rejected with 503 and the event is logged as WARNING, not CRITICAL (not a confirmed infection)

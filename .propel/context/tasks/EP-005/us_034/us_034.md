# User Story - US_034

## Story ID
* US_034

## Story Title
* Staff same-day walk-in queue view and manual reorder

## Description
* As a Staff member, I want to view the current same-day walk-in queue with estimated wait times and manually reorder patients when clinical priority changes, so that I can manage dynamic queue situations without a separate workflow system.

## Acceptance Criteria

### AC-001: GET queue returns all entries with estimated wait times
- **Given**: The same-day walk-in queue contains entries
- **When**: `GET /staff/queue` is called by an authenticated Staff member
- **Then**: HTTP 200 is returned with an ordered list of `{position, patientId, name, estimatedWaitMinutes, status, addedAt}`; `estimatedWaitMinutes` is calculated as `position × avgServiceTimeMinutes`

### AC-002: Reorder changes position and recalculates wait times
- **Given**: Staff calls `PATCH /staff/queue/reorder` with a new ordered list of patient IDs
- **When**: The reorder is processed
- **Then**: HTTP 200 is returned with the updated queue order; `estimatedWaitMinutes` is recalculated for all entries; an AuditLog entry records the reorder with before/after position data

### AC-003: Concurrent queue edit returns 409 with a refresh hint
- **Given**: Two Staff members simultaneously call `PATCH /staff/queue/reorder`
- **When**: The second request attempts to commit while the first is in progress
- **Then**: HTTP 409 is returned with `{"error":"Queue updated by another Staff member","hint":"Refresh queue and retry"}`; the second reorder is not committed

### AC-004: Entry removal from queue (patient leaves without being seen)
- **Given**: A patient in the queue leaves without being seen
- **When**: `DELETE /staff/queue/{entryId}` is called
- **Then**: HTTP 200 is returned; the QueueEntry is removed; the queue is re-indexed; remaining estimated wait times are recalculated

## Edge Cases
* Empty queue — `GET /staff/queue` returns HTTP 200 with an empty array, not 404
* Reorder with a missing patient ID — validation rejects the reorder if any submitted ID is not in the current day's queue; HTTP 422 returned
* Average service time changes during the day — `avgServiceTimeMinutes` is a configurable value from `AppSettings`; updated service time affects all subsequent wait time calculations

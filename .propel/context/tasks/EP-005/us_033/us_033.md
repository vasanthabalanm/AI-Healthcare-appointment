# User Story - US_033

## Story ID
* US_033

## Story Title
* Staff walk-in patient registration and queue addition

## Description
* As a Staff member, I want to search for an existing patient or create a minimal walk-in profile and add them to the same-day queue, so that unscheduled patients receive care without disrupting the pre-booked appointment flow.

## Acceptance Criteria

### AC-001: Search returns matching patients by name or DOB
- **Given**: Staff calls `GET /staff/patients/search?q=Jones&dob=1985-04-12`
- **When**: The query runs against UserAccounts
- **Then**: HTTP 200 is returned with a list of matching patients (max 20 results); each result includes `{patientId, fullName, dob, lastAppointmentDate}`

### AC-002: Existing patient added to walk-in queue returns 201
- **Given**: A patient was found via search and the queue has capacity
- **When**: `POST /staff/queue {patientId, priority:"Normal"}` is called
- **Then**: HTTP 201 is returned; a QueueEntry is inserted for today's queue; `estimatedWaitMinutes` is calculated based on current queue position

### AC-003: Walk-in queue at capacity returns 409
- **Given**: The walk-in queue has reached its configured daily maximum (default 20 entries)
- **When**: `POST /staff/queue {patientId}` is called
- **Then**: HTTP 409 is returned with `{"error":"Walk-in queue at capacity","hint":"Override with priority flag"}`; no entry is created

### AC-004: Capacity override flag adds patient despite queue limit
- **Given**: The queue is at capacity
- **When**: `POST /staff/queue {patientId, priority:"Override"}` is called
- **Then**: HTTP 201 is returned; the patient is added to the queue with a visual override indicator; an AuditLog entry records the override with the Staff actor

### AC-005: Minimal walk-in profile created for unknown patient
- **Given**: No matching patient is found in the search results
- **When**: `POST /staff/patients/walkin {name, dob, phone}` is followed by `POST /staff/queue {newPatientId}`
- **Then**: A new UserAccount is created with `role=Patient`, `status=Active`, `emailVerified=false`, `walkin=true`; the patient is added to the queue

## Edge Cases
* Staff searches with only one character — minimum 3-character search enforced; HTTP 422 returned if query < 3 characters
* Walk-in patient already in queue for today — duplicate queue entry is prevented; HTTP 409 returned with "Patient already in today's queue"
* System clock at midnight — "today's queue" is defined by UTC date; rollover handled gracefully with no orphaned queue entries from the prior day

# User Story - US_035

## Story ID
* US_035

## Story Title
* Staff patient arrival check-in

## Description
* As a Staff member, I want to check in a patient on arrival so that their appointment status is updated to Arrived and they are automatically removed from the walk-in queue, giving the care team a real-time view of who is present.

## Acceptance Criteria

### AC-001: Successful check-in sets appointment status to Arrived
- **Given**: An appointment with `status=Scheduled` exists for today
- **When**: `PATCH /appointments/{id}/checkin` is called by an authenticated Staff member
- **Then**: HTTP 200 is returned; the Appointment `status` is set to `Arrived`; `arrivedAt` is set to UtcNow; any matching QueueEntry for the patient is removed from the walk-in queue

### AC-002: Check-in writes an AuditLog entry
- **Given**: Staff checks in a patient
- **When**: `PATCH /appointments/{id}/checkin` is processed
- **Then**: An AuditLog entry is written with `actionType=UPDATE`, `entityType=Appointment`, `beforeValue={"status":"Scheduled"}`, `afterValue={"status":"Arrived"}`, `actorId=staffId`

### AC-003: Patient already marked Arrived returns 409 with override option
- **Given**: The appointment already has `status=Arrived`
- **When**: `PATCH /appointments/{id}/checkin` is called again
- **Then**: HTTP 409 is returned with `{"error":"Patient already checked in","hint":"Set override:true to re-confirm"}`; no status change is made unless the request includes `{override:true}`

### AC-004: Check-in of an appointment not found returns 404 with walk-in hint
- **Given**: The `appointmentId` in the path does not exist in the database
- **When**: `PATCH /appointments/{id}/checkin` is called
- **Then**: HTTP 404 is returned with `{"error":"Appointment not found","hint":"Register as walk-in if unscheduled"}`

## Edge Cases
* Patient checks in for a future-dated appointment — check-in allowed only for today's appointments; HTTP 409 returned if appointment date is not today
* Check-in race condition (two Staff members scanning same patient simultaneously) — EF Core rowversion lock prevents double status update; the second request receives 409
* Patient's appointment is Cancelled — check-in returns HTTP 409 `{"error":"Cannot check in — appointment is Cancelled"}`

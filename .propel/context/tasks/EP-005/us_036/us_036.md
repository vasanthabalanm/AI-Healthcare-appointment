# User Story - US_036

## Story ID
* US_036

## Story Title
* Staff daily appointment schedule view

## Description
* As a Staff member, I want to view all appointments for today with their current status and risk indicators in a single screen, so that I can manage the day's patient flow efficiently without switching between multiple views.

## Acceptance Criteria

### AC-001: Daily schedule returns all today's appointments with status
- **Given**: An authenticated Staff member calls `GET /schedule/today`
- **When**: The query runs against Appointments with `slotDate = today`
- **Then**: HTTP 200 is returned with an ordered list of `{appointmentId, patientName, slotTime, status, noShowRiskScore, riskFlag, intakeStatus}`; the list is ordered by `slotTime ASC`

### AC-002: Schedule response includes intake completion status per appointment
- **Given**: Some appointments have a submitted IntakeRecord and some do not
- **When**: `GET /schedule/today` is called
- **Then**: Each appointment entry includes `intakeStatus` as one of `Submitted`, `Pending`, or `NA` (for walk-ins); appointments without intake are visually flagged

### AC-003: Risk flag is present for high-risk appointments
- **Given**: An appointment has `noShowRiskScore ≥ configured threshold (default 70)`
- **When**: The schedule is loaded
- **Then**: The appointment entry includes `riskFlag=true`; no-risk appointments have `riskFlag=false`

### AC-004: Empty today schedule returns 200 with empty array
- **Given**: No appointments are scheduled for today
- **When**: `GET /schedule/today` is called
- **Then**: HTTP 200 is returned with `{"appointments":[], "date":"<today>"}`; 404 is not returned

## Edge Cases
* Appointment crosses midnight (edge case slot time = 23:45 but care extends to next day) — appointment is included in the schedule for the slot's start date only
* Staff calls schedule for a different date (historical or future) — a `?date=YYYY-MM-DD` parameter is supported; same query logic applies; future dates show only booked appointments
* Schedule includes ≥ 100 appointments — response is paginated with max 50 per page; `?page=2` parameter is supported

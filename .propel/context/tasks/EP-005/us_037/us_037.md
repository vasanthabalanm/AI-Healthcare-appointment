# User Story - US_037

## Story ID
* US_037

## Story Title
* No-show risk alerts and proactive outreach recording

## Description
* As a Staff member, I want to see a filtered list of high-risk appointments and record proactive outreach attempts, so that I can reduce no-show rates by contacting at-risk patients before their appointment.

## Acceptance Criteria

### AC-001: High-risk appointment filter returns only appointments above threshold
- **Given**: Appointments for upcoming dates include both low-risk and high-risk entries
- **When**: `GET /schedule/high-risk?date=<date>` is called by Staff
- **Then**: HTTP 200 is returned with only appointments where `riskFlag=true` (score ≥ threshold); low-risk appointments are excluded; list is ordered by `noShowRiskScore DESC`

### AC-002: Staff records an outreach attempt on a high-risk appointment
- **Given**: An appointment is flagged as high-risk
- **When**: `POST /appointments/{id}/outreach {note:"Called patient — confirmed attendance"}` is called by Staff
- **Then**: HTTP 201 is returned; an OutreachNote is saved with `staffId`, `appointmentId`, `note`, `timestamp`; the appointment entry in the schedule reflects that outreach was recorded

### AC-003: Staff marks appointment as NoShow after non-attendance
- **Given**: The appointment time has passed and the patient did not arrive
- **When**: `PATCH /appointments/{id}/status {status:"NoShow"}` is called
- **Then**: HTTP 200 is returned; Appointment `status` is set to `NoShow`; the slot is released (`isAvailable=true`); an AuditLog entry is written; the no-show count metric for the patient is incremented

### AC-004: Empty high-risk list returns 200 with empty array and message
- **Given**: No appointments for the date meet the high-risk threshold
- **When**: `GET /schedule/high-risk?date=<date>` is called
- **Then**: HTTP 200 is returned with `{"appointments":[], "message":"No high-risk appointments for this date"}`

## Edge Cases
* Threshold changed after appointments already flagged — the `riskFlag` is re-evaluated at query time against the current threshold, not stored; changing the threshold immediately affects filter results
* Staff records outreach on a non-high-risk appointment — outreach can be recorded on any appointment; the endpoint does not restrict to high-risk only; the risk filter is a view concern only
* NoShow status applied to an appointment already in Arrived or InProgress — the FSM interceptor rejects the transition; HTTP 409 returned with valid next states

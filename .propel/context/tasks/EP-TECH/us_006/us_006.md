# User Story - US_006

## Story ID
* US_006

## Story Title
* Serilog structured logging, Seq integration, and OSS licence audit

## Description
* As a developer, I want Serilog configured with rolling file and Seq Community Edition sinks, correlation ID middleware, and a PHI-safe log filter, and all project dependencies audited for OSS licence compliance, so that every log entry is queryable and traceable and the platform remains free of paid-licence components.

## Acceptance Criteria

### AC-001: Serilog rolling file sink creates daily log files with 30-day retention
- **Given**: Serilog is configured with `RollingInterval.Day` and `retainedFileCountLimit: 30`
- **When**: The application runs for 24 hours past midnight
- **Then**: A new log file is created with a date-stamped filename; log files older than 30 days are automatically deleted; no unhandled IO exceptions are raised during rotation

### AC-002: Seq Community Edition sink receives structured log events
- **Given**: The `Seq:ServerUrl` environment variable points to a running Seq CE instance
- **When**: Any API request is processed
- **Then**: The request's log event appears in Seq within 5 seconds with structured properties (CorrelationId, UserId, RequestPath, StatusCode); the event is queryable via Seq's filter syntax

### AC-003: Correlation ID middleware injects a unique ID per request
- **Given**: An HTTP request arrives without an `X-Correlation-ID` header
- **When**: The middleware processes the request
- **Then**: A new GUID is generated and attached to the request context; all log events for that request include `CorrelationId` = the generated GUID; the `X-Correlation-ID` is echoed in the response header

### AC-004: PHI filter prevents any PHI from appearing in log output
- **Given**: A log statement includes a field named `PatientName`, `DateOfBirth`, `Email`, or `PasswordHash`
- **When**: The Serilog destructuring policy and the PHI filter are applied
- **Then**: The field is replaced with `[REDACTED]` in the log output; no raw PHI value appears in rolling file or Seq output

### AC-005: All NuGet and npm dependencies are confirmed OSS-licensed
- **Given**: A dependency audit script reads `*.csproj` and `package.json` files
- **When**: The script runs against all packages in the solution
- **Then**: Every package is classified as MIT, Apache 2.0, LGPL, or GPL; any package not matching these licences causes the audit to exit with a non-zero code and lists the offending package

## Edge Cases
* Seq CE unavailable (network partition) — Serilog should buffer events in memory (up to configured limit) and fall back to file-only logging without crashing; reconnection is automatic
* Log volume spike causing disk pressure — rolling file sink's retention policy and a disk-space guard log warning at 80% capacity
* Third-party package releases a new version with a changed licence — CI OSS audit runs on every dependency update PR to catch reclassification early

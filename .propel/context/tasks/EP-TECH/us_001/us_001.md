# User Story - US_001

## Story ID
* US_001

## Story Title
* Angular 17 SPA scaffold with RBAC route guards

## Description
* As a developer, I want the Angular 17 SPA project scaffolded with role-based route guards and a role-aware layout shell, so that all future feature modules can be added to a consistent, secure, and deployable frontend foundation.

## Acceptance Criteria

### AC-001: Angular project initialises and builds successfully
- **Given**: The repository is freshly cloned and `npm install` has completed
- **When**: `ng build --configuration production` is executed
- **Then**: The build completes with zero errors, producing a `dist/` folder with bundled HTML, CSS, and JS artefacts

### AC-002: RBAC route guards block unauthorised navigation
- **Given**: A user is not authenticated (no JWT in local storage)
- **When**: The user navigates to a protected route (e.g., `/staff/queue`)
- **Then**: The `AuthGuard` redirects the browser to `/login?reason=unauthorized` and the protected component is never rendered

### AC-003: Cross-role navigation is blocked by RoleGuard
- **Given**: A Patient-role JWT is present in local storage
- **When**: The user navigates to `/staff/schedule`
- **Then**: The `RoleGuard` redirects the browser to `/login?reason=unauthorized` and logs the violation

### AC-004: Role-aware navigation shell renders correct menu per role
- **Given**: A Staff-role JWT is present and the user is authenticated
- **When**: The layout shell renders
- **Then**: Only Staff-facing navigation links are visible; Patient-facing and Admin-facing links are absent from the DOM

### AC-005: HTTPS enforced in Angular production build
- **Given**: The Angular environment file is configured for production
- **When**: The app is served from Netlify or Vercel
- **Then**: All API calls in the HTTP client base URL use `https://`; no `http://` origins are present in the production bundle

### AC-006: Netlify/Vercel deploy configuration is present and valid
- **Given**: The repository contains `netlify.toml` or `vercel.json`
- **When**: A CI/CD push to main branch triggers a deploy
- **Then**: The static site deploys successfully with SPA redirect rules (`/*` → `index.html`) so deep-linking works

## Edge Cases
* Empty JWT payload with valid signature — `AuthGuard` must reject and redirect; guard must not throw an unhandled exception
* Route navigation to an unknown path — wildcard `**` route redirects to role-appropriate dashboard (not a blank screen)
* JWT expiry during client-side navigation — `AuthGuard` detects expired token before API call and redirects to `/login?reason=timeout`
* Simultaneous role change by Admin while user is logged in — next route activation re-reads token and applies latest role permissions

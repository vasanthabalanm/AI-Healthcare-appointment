# User Story - US_005

## Story ID
* US_005

## Story Title
* Deployment configuration for IIS, Netlify, and Windows Services

## Description
* As a developer, I want deployment manifests and configuration for all three deployment targets — Angular SPA to Netlify/Vercel, .NET 8 API to IIS, and Hangfire/Ollama/Rasa as Windows Services — so that every component can be deployed and run in production without manual configuration steps.

## Acceptance Criteria

### AC-001: Angular production build deploys to Netlify with SPA routing
- **Given**: A `netlify.toml` (or `vercel.json`) file exists in the repository root
- **When**: The CI/CD pipeline pushes a production Angular build to Netlify/Vercel
- **Then**: The site is accessible over HTTPS; navigating to any deep-link path (e.g., `/staff/queue`) returns the Angular `index.html` (not a 404); HSTS header is present in the response

### AC-002: .NET 8 API publishes as IIS in-process hosting
- **Given**: A `web.config` with `<aspNetCore processPath="dotnet" hostingModel="inprocess" />` is present
- **When**: `dotnet publish -c Release` is executed and the output is deployed to an IIS site
- **Then**: The IIS application pool starts, the API responds to `GET /health` with HTTP 200, and the process name is `w3wp.exe` (confirming in-process hosting)

### AC-003: Hangfire Windows Service runs and processes jobs
- **Given**: A `WindowsServiceInstaller` (via `UseWindowsService()`) is configured in the Hangfire worker project
- **When**: The service is installed via `sc create` and started
- **Then**: The Windows Service status shows "Running"; Hangfire picks up and processes a test job within 30 seconds

### AC-004: TLS certificate is configured for IIS HTTPS binding
- **Given**: A self-signed or CA-issued TLS certificate is bound to the IIS site on port 443
- **When**: A browser navigates to the API base URL over HTTP
- **Then**: IIS redirects to HTTPS (301); the TLS handshake completes; the API responds to `GET /health`

### AC-005: Environment variable secrets are not present in any committed configuration file
- **Given**: All `appsettings.json` and `appsettings.Production.json` files are inspected
- **When**: Searching for connection strings, API keys, or secrets
- **Then**: No credentials are found; all sensitive values reference environment variable placeholders (e.g., `${DB_CONNECTION_STRING}`); `.gitignore` excludes any `.env` file

## Edge Cases
* IIS application pool recycle mid-request — in-process hosting model means recycle kills in-flight requests; ensure graceful shutdown (`UseShutdownTimeout`) is configured to 30 seconds
* Netlify deploy fails due to build output path mismatch — `publish` directory in `netlify.toml` must match `outputPath` in `angular.json`; CI pipeline validates this before deploy
* Windows Service fails to start due to missing .NET 8 runtime — installer script must check for runtime presence and emit a clear error message
